# OSSTEM Desktop App 기능 기획

## 1. 앱 실행

시스템 트레이에 상주하는 데스크톱 앱이다. denall.com/osstem.com 웹 서비스를 래핑한다.

### 첫 실행

앱을 처음 설치하면 시스템 시작 시 자동 실행이 활성화된다.

### 중복 실행 방지

앱이 이미 실행 중일 때 다시 실행하면 새 인스턴스를 띄우지 않고 기존 앱을 활성화한다.

- 로그인 상태: 메인 창을 복원하고 포커스한다.
- 비로그인 상태: 로그인 창을 표시한다.

### 종료

모든 창을 닫아도 앱은 종료되지 않고 트레이에 남는다. 트레이 메뉴의 "종료"를 선택해야 완전히 종료된다.

---

## 2. 로그인

SSO 기반 로그인으로 앱을 사용한다.

### 로그인 창

- 웹 기반 SSO 로그인 페이지를 표시한다.
- 뒤로가기 버튼: 이전 페이지가 있을 때만 활성화된다.
- 로그인 버튼: SSO 로그인 페이지로 돌아간다.

### 로그인 완료

로그인에 성공하면 다음이 순서대로 수행된다.

1. 로그인 창이 닫힌다.
2. 메인 창이 표시된다.
3. 사용자 프로필을 조회한다.
4. 채팅 메시지 수신을 위한 실시간 연결을 시작한다.
5. 트레이 메뉴가 로그인 상태로 전환된다.

---

## 3. 로그아웃

트레이 메뉴에서 로그아웃할 수 있다.

### 로그아웃 흐름

1. 읽지 않은 메시지 배지를 초기화한다.
2. 실시간 연결을 해제한다.
3. 서버 세션을 제거한다.
4. 모든 열린 창을 닫는다.
5. 로그인 창을 표시한다.

---

## 4. 로그인 유지

세션이 만료되어도 자동으로 갱신하여 로그인 상태를 유지한다.

### 동작 조건

로그인 시 "로그인 유지" 옵션을 선택한 경우에만 동작한다.

### 갱신 흐름

- 세션이 만료(12시간)되면 자동으로 토큰 갱신을 시도한다.
- 갱신에 성공하면 사용자는 아무 변화를 느끼지 못하고 계속 사용할 수 있다.
- 갱신에 실패하면 자동으로 로그아웃된다.

### "로그인 유지" 미선택 시

세션이 만료되면 즉시 로그아웃되고 로그인 창이 표시된다.

---

## 5. 메인 창

로그인 후 표시되는 앱의 주 화면이다.

### 레이아웃

좌측 사이드바와 우측 콘텐츠 영역으로 구성된다.

### 사이드바 메뉴

5개의 아이콘 버튼으로 구성되며, 각 버튼에 툴팁이 표시된다.

- **BOOK** — 예약 관리
- **CRM** — 고객 관리
- **POST** — 메시지 (기본 선택)
- **JOB** — 채용
- **MY PAGE** — 내 정보

메뉴를 클릭하면 콘텐츠 영역이 해당 서비스 화면으로 전환된다.

### 닫기 동작

X 버튼을 누르면 창을 닫지 않고 숨긴다. 트레이에서 다시 열 수 있다.

---

## 6. 대화방 창

특정 대화방을 별도 창으로 열어 대화할 수 있다.

### 열리는 경우

- 토스트 알림을 클릭한 경우
- 웹 서비스에서 대화방 열기를 요청한 경우

### 동작

- 대화방 하나당 하나의 창만 열린다.
- 이미 열려 있으면 해당 창을 포커스한다.
- 로그아웃 시 모든 대화방 창이 닫힌다.

---

## 7. 토스트 알림

새 채팅 메시지가 도착하면 화면 우하단에 토스트 알림을 표시한다.

### 표시 조건

메시지가 도착하면 토스트를 표시한다. 단, 다음의 경우에는 표시하지 않는다.

- 환경설정에서 토스트 알림이 꺼져 있는 경우
- 해당 대화방 창이 열려 있고 포커스 상태인 경우

토스트가 표시되지 않더라도 작업표시줄 깜빡임과 배지는 정상 동작한다.

### 표시 내용

- 발신자 아바타 (이름 첫 글자, 주황색 원형 배경)
- 발신자 이름
- 메시지 내용
- 발신 시각 (HH:mm)

### 표시 위치

화면 우하단에 표시된다. 여러 개의 토스트가 동시에 표시되면 아래에서 위로 쌓인다.

### 자동 닫힘

5초 후 자동으로 닫힌다.

### 동일 대화방 메시지

같은 대화방에서 연속으로 메시지가 오면 기존 토스트의 내용을 갱신하고 5초 타이머를 재시작한다. 새 토스트를 추가로 띄우지 않는다.

### 클릭 동작

토스트를 클릭하면 해당 대화방 창이 열리고, 깜빡임과 배지가 초기화된다.

### 닫힌 후

토스트가 닫히면 나머지 토스트의 위치를 자동 재정렬한다.

### 알림음

토스트가 표시될 때 알림음을 재생한다.

### 환경설정

- 토스트 알림 표시: on/off (기본값: on)
- 알림 소리 재생: on/off (기본값: on)

---

## 8. 작업표시줄 깜빡임

새 채팅 메시지가 도착하면 작업표시줄 아이콘을 깜빡여 사용자의 주의를 끈다.

### 깜빡임 조건

메시지가 도착하면 해당 창의 작업표시줄 아이콘을 깜빡인다. 단, 다음의 경우에는 깜빡이지 않는다.

- 해당 대화방 창이 포커스 상태인 경우

### 깜빡임 대상 창

- 대화방 창이 열려 있는 경우 — 해당 대화방 창을 깜빡인다.
- 대화방 창이 없는 경우 — 메인 창을 최소화 상태로 띄운 뒤 메인 창을 깜빡인다.

### 깜빡임 방식

0.5초 간격으로 3회 깜빡인 뒤 자동으로 멈춘다.

### 깜빡임 해제

다음의 경우 깜빡임을 즉시 해제한다.

- 메인 창이 포커스를 받은 경우
- 대화방 창이 포커스를 받은 경우
- 토스트 알림을 클릭하여 대화방을 연 경우

---

## 9. 배지 (읽지 않은 메시지 수)

작업표시줄 아이콘에 읽지 않은 메시지 수를 배지로 표시한다.

### 카운트 증가

메시지 도착 시 카운트를 1 증가시킨다.

### 카운트 초기화

다음의 경우 카운트를 0으로 초기화한다.

- 메인 창 또는 대화방 창이 포커스를 받은 경우
- 토스트 클릭으로 대화방을 연 경우
- 로그아웃한 경우

### 표시 형식

99개 초과 시 "99+"로 표시한다. 트레이 툴팁에 "OSSTEM (N개의 새 메시지)"로 표시한다.

---

## 10. 본인 메시지 필터링

기업 회원인 경우, 본인이 보낸 메시지에 대해서는 알림하지 않는다.

### 동작 조건

- 현재 사용자가 기업 회원이고
- 발신자가 본인인 경우

해당 메시지에 대해 토스트, 깜빡임, 배지 모두 동작하지 않는다.

---

## 11. 시스템 트레이

앱은 시스템 트레이에 상주한다.

### 더블클릭

- 로그인 상태: 메인 창을 표시하고 포커스한다.
- 비로그인 상태: 로그인 창을 표시한다.

### 우클릭 메뉴 (로그인 상태)

1. 열기
2. 환경설정
3. 로그아웃
4. 종료

### 우클릭 메뉴 (비로그인 상태)

1. 로그인
2. 환경설정
3. 종료

---

## 12. 환경설정

앱 동작을 사용자가 설정할 수 있다. 트레이 메뉴의 "환경설정"으로 연다.

### 일반

| 항목 | 기본값 | 설명 |
|------|--------|------|
| 시스템 시작 시 자동 실행 | on (첫 설치 시) | Windows 시작 시 앱 자동 실행 |

### 알림

| 항목 | 기본값 | 설명 |
|------|--------|------|
| 토스트 알림 표시 | on | 끄면 토스트가 표시되지 않음 (깜빡임, 배지는 유지) |
| 알림 소리 재생 | on | 끄면 알림음이 재생되지 않음 |

---

## 13. 외부 링크 처리

앱 내에서 외부 URL을 클릭하면 시스템 기본 브라우저에서 열린다. denall.com과 osstem.com 도메인은 앱 내에서 처리한다.
