# 오스템 데스크톱 앱 기획

## 개요
osstem.com 서브도메인 사이트들을 Electron 기반 데스크톱 앱으로 제공한다.

## 기술 스택
- **Electron** (웹사이트를 데스크톱 앱으로 래핑)

## 대상 사이트

| 사이트 | URL | 용도 |
|--------|-----|------|
| 통합회원 | `https://member.denall.com/` | 로그인, 회원가입, 계정정보 변경 |
| 오스템 채팅 | `https://mcs.osstem.com/desktop/post` | 메인 채팅 화면 |

## 앱 실행 흐름

1. 앱 시작
2. 로그인 상태 확인
   - **로그인 됨** → 오스템 채팅 화면으로 이동
   - **로그인 안 됨** → 통합회원 로그인 화면 표시 → 로그인 완료 후 오스템 채팅으로 이동
3. 로그인 세션 유지 시간이 길어 대부분 로그인 상태가 유지됨

## 시스템 트레이

- 앱은 시스템 트레이에 **상주**
- **더블클릭** → 오스템 채팅 창 열기
- **우클릭** → 컨텍스트 메뉴 표시
- **X 버튼** → 창만 숨기고 트레이에 계속 상주 (종료하지 않음)

### 트레이 컨텍스트 메뉴 (로그인 상태별)

| 메뉴 항목 | 비로그인 | 로그인 |
|-----------|:---:|:---:|
| 로그인 | O | X |
| 오스템 채팅 열기 | X | O |
| 계정정보 변경 | X | O |
| 종료 | O | O |

## 웹뷰 설정

- User-Agent에 `osstem-app:1.0.0` 강제 삽입
- 채팅 웹 서비스는 User-Agent로 데스크톱 앱 여부를 판별

## 알림

- **형태**: 커스텀 디자인 토스트 (우측 하단)
- **트리거**: 오스템 채팅에서 새 메시지 수신 시

### 알림 흐름

```
채팅 웹 서비스 (웹소켓으로 메시지 수신)
  → User-Agent로 앱 환경 판별
  → 래핑된 인터페이스 호출 (IPC를 직접 모름)
  → Electron IPC 통신
  → 커스텀 토스트 알림 표시
```

## 통신 구조 (IPC 래핑)

- Electron ↔ 웹페이지 간 통신은 **IPC** 사용
- 채팅 웹 서비스 코드에서는 IPC를 직접 알지 못하도록 **래핑**
- preload script에서 래핑 인터페이스를 `window` 객체에 노출

```
// 채팅 웹 서비스 쪽 (IPC를 모름)
window.osstemApp.notify({ sender: '홍길동', message: '안녕하세요' });

// 내부적으로
// preload: ipcRenderer.send(...) → main process → 토스트 표시
```

## 미결정 사항

- [ ] 래핑 인터페이스 네이밍 (예: `window.osstemApp`)
- [ ] 알림 전달 데이터 항목 (보낸 사람, 메시지 내용, 채팅방 정보 등)
- [ ] 커스텀 토스트 디자인 및 클릭 시 동작
- [ ] 사용자 범위 (본인용 vs 사내 배포)
- [ ] 앱 버전 관리 및 자동 업데이트 여부
